# Constant-Rate Core

Up: [`index.md`](index.md)

This page documents the single-type constant-rate birth-death-sampling core.
It introduces the process parameters, simulation event logs, and the basic
count queries used by the original-process and reconstructed-process pages.

Related pages:

- [`original_process_counts.md`](original_process_counts.md)
- [`reconstructed_process.md`](reconstructed_process.md)

## Shared Notation

Times are forward process times.

- `t_i`: start time for an analytical interval.
- `t_j`: query time.
- `t_k`: observation or truncation horizon, with `t_i <= t_j <= t_k`.
- `N(t)`: number of active lineages at time `t`.
- `S(t)`: cumulative number of sampled lineages observed up to time `t`.
- `A(t)`: number of lineages active at `t` that are retained by a later sample
  in `(t, t_k]`.

> Original vs reconstructed: `N(t)` and `S(t)` describe the original simulated
> process. `A(t)` describes the reconstructed process after pruning away
> lineages that do not lead to later observations.

## Parameters

`ConstantRateBDParameters(λ, μ, ψ, r, ρ₀=0.0)` stores the single-type
constant-rate process:

- `λ`: birth rate per active lineage.
- `μ`: death rate per active lineage.
- `ψ`: sampling rate per active lineage.
- `r`: probability that a sampling event removes the sampled lineage.
- `ρ₀`: present-day sampling probability applied to lineages still active at
  the simulation horizon when `apply_ρ₀=true`.

The constructor checks the currently supported domain: `λ > 0`, `μ >= 0`,
`ψ >= 0`, and `r, ρ₀ in [0, 1]`.

## Event Types

`simulate_bd` records four event kinds in a `BDEventLog`.

- `Birth`: one active lineage gives birth to a new active child lineage. The
  parent remains active.
- `Death`: one active lineage dies without being sampled.
- `FossilizedSampling`: one active lineage is sampled but not removed. This
  increases `S(t)` but leaves `N(t)` unchanged.
- `SerialSampling`: one active lineage is sampled and removed. This increases
  `S(t)` and decreases `N(t)`.

Present-day samples generated by `ρ₀` are recorded as `SerialSampling` events at
`tmax`.

## Simulation And Event Logs

Use `simulate_bd([rng], pars, tmax; initial_lineages=1, apply_ρ₀=true)` to
simulate forward from time `0` to `tmax`.

The returned `BDEventLog` is a struct-of-arrays event history with:

- `time`: event times.
- `lineage`: affected lineage id.
- `parent`: parent id for births, or `0` otherwise.
- `kind`: one of the event kinds above.
- `initial_lineages`: number of lineages active at time `0`.
- `tmax`: simulation horizon.

The log records process events, not a tree likelihood object. It preserves
enough lineage and parent information for count queries and tree extraction
helpers.

## Basic Queries

Use `NS_at(log, t)` to get `(N=N(t), S=S(t))` at one time. `N_at` and `S_at`
return the individual components. The `*_over_time` variants evaluate the same
queries over a vector of times.

For reconstructed counts, `A_at(log, t_j, t_k=log.tmax)` counts active lineages
at `t_j` that have at least one sampled descendant in `(t_j, t_k]`. See
[`reconstructed_process.md`](reconstructed_process.md) for the reconstructed
interpretation.

## Minimal Example

```julia
using Random
using BDUtils

rng = MersenneTwister(20260420)
pars = ConstantRateBDParameters(1.2, 0.4, 0.6, 0.5, 0.25)

log = simulate_bd(rng, pars, 2.0; initial_lineages=1, apply_ρ₀=true)

println(log)
println("events = ", collect(log))
println("N(1.0), S(1.0) = ", NS_at(log, 1.0))
println("N over time = ", N_over_time(log, [0.0, 1.0, 2.0]))
println("S over time = ", S_over_time(log, [0.0, 1.0, 2.0]))
println("A(1.0) with horizon 2.0 = ", A_at(log, 1.0, 2.0))
```

The exact events depend on the random seed, but the interpretation is fixed:
births add active lineages, deaths remove unobserved lineages, fossilized
samples add observations without removal, and serial samples add observations
with removal.
